Book Rendering Engine — Spec v1.0
Goal:
Play NBF books smoothly, securely, and beautifully in any modern browser.
________________________________________
1️⃣ Reader App Responsibilities
The Reader App must:
1.	Open encrypted NBF books securely 
2.	Render HTML story beautifully 
3.	Synchronize text + audio perfectly 
4.	Provide reader personalization controls 
5.	Prevent content piracy 
6.	Work on mobile, tablet, desktop 
This is both a reader UI + media engine.
________________________________________
2️⃣ High Level Architecture
User Browser
   │
   ├── UI Layer (Reader Interface)
   ├── Rendering Engine (HTML Renderer)
   ├── Audio Engine (Web Audio API)
   ├── Timeline Sync Engine
   ├── DRM Security Layer
   └── Streaming & Decryption Layer
We split into 5 core engines.
________________________________________
3️⃣ Secure Book Loading Flow 🔐
Step-by-step opening a book
1.	User clicks book in library 
2.	Browser requests book session token 
3.	Backend returns: 
o	Encrypted production.json 
o	Signed audio URLs 
4.	Book decrypted in memory only 
5.	Reader starts playback 
Security rule:
👉 No permanent files stored on device.
________________________________________
4️⃣ Book Loader Engine 📦
Responsibilities:
•	Download manifest.json 
•	Load story.html into sandbox iframe 
•	Fetch encrypted production.json 
•	Initialize character + timeline data 
Important:
The story HTML runs inside a sandboxed iframe to prevent copying.
________________________________________
5️⃣ HTML Rendering Engine 🖥️
This engine converts story tags into UI.
Custom tags → React/Vue components (internally).
Tag	Rendered As
<narration>	Paragraph block
<dialogue>	Speech bubble
<thought>	Italic block
<quote>	Center block
<pause>	Invisible timing marker
________________________________________
6️⃣ Audio Engine 🎧 (Core Technology)
Uses Web Audio API.
Why?
Because we need real-time mixing of multiple audio layers.
Audio Channels
Channel	Example
Narrator	Voice track
Characters	Voice track
Music	Loop track
Ambience	Loop track
SFX	Triggered track
All channels have independent volume sliders.
________________________________________
7️⃣ Timeline Sync Engine ⏱️ (MAGIC)
This engine keeps text highlight synced with audio.
Workflow:
1.	Load timeline from production.json 
2.	Play narration audio 
3.	Highlight block by ID when audio starts 
4.	Auto-scroll to active block 
Reader feels like text is being read live.
________________________________________
8️⃣ Text Highlight System ✨
When audio plays:
•	Current block glows softly 
•	Previous blocks fade 
•	Page auto-scrolls smoothly 
This gives a karaoke reading experience.
________________________________________
9️⃣ Reader Modes Implementation
📖 Reading Mode
•	Audio optional 
•	Manual scroll 
•	Highlight disabled by default 
🎧 Audiobook Mode
•	Auto-scroll ON 
•	Highlight ON 
•	Music OFF by default 
🎬 Cinematic Mode
•	Fullscreen 
•	Music + ambience ON 
•	Larger text 
•	Slower pacing 
Modes only change engine settings, not content.
________________________________________
🔟 Reader Control Panel 🎛️
Floating bottom panel:
Audio Controls

•	Narrator volume

•	Music volume 
•	SFX volume 
•	Playback speed 
Reading Controls
•	Font size

Preferences saved per user.
________________________________________
11️⃣ Resume & Bookmark System 🔖
Reader App saves:
•	Last block read 
•	Playback timestamp 
•	Reader preferences 
When user reopens book:
👉 Resume instantly.
________________________________________
12️⃣ Streaming & Performance ⚡
We stream audio per scene.
Why?
Books may contain 100+ audio files.
Engine loads:
•	Current scene audio 
•	Preloads next scene audio 
This ensures:
✔ Fast loading
✔ Low bandwidth
✔ Smooth playback

13️⃣ DRM Protection Layer 🛡️
Anti-piracy features:
•	Sandboxed HTML iframe 
•	Disable right-click & text selection 
•	Obfuscated DOM structure 
•	Streaming signed URLs 
•	Audio chunks (not full files) 
•	Per-user watermark injection 
Goal: prevent casual piracy.
