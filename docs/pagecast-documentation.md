## 0. Executive Overview

### 0.1 Vision of PageCast

PageCast is a new generation of digital storytelling — not an ebook, not an audiobook, not a website with audio. It is a cinematic storytelling platform where creators publish interactive HTML stories that readers can read and hear simultaneously, directly in any modern browser. No apps. No downloads. Just storytelling brought to life.

Think of it as:
- **Netflix narration quality** 🎧
- **Kindle reading simplicity** 📖
- **Web technology flexibility** 🌐

Readers open a link → enter a secure reader → read and listen simultaneously. This is powerful.

### 0.2 Problem Statement (Traditional Storytelling Limitations)

Traditional digital formats each carry a fundamental limitation:

| Format | Problem |
|--------|---------|
| PDF | Static, no audio sync |
| EPUB | Limited multimedia support |
| Audiobook | No text — imagination unsupported |
| Website stories | Not sellable / not protected |

Books are powerful — but limited. No voice. No emotion. No sound. No atmosphere. Audiobooks solve sound but lose text. Movies solve visuals but remove imagination. Something is missing.

### 0.3 Proposed Solution (Cinematic Voice + Text Storytelling Platform)

PageCast combines all four dimensions into one seamless experience:

> **Text + Audio + Formatting + Commerce**

The PageCast reader experience delivers:
- Narrator voice
- Character voices
- Background music
- Sound effects
- Highlighted text while audio plays
- Cinematic reading mode

This is closer to interactive storytelling than traditional eBooks. It makes every story a mini movie in the browser.

### 0.4 Product Summary (Reader + Creator Ecosystem)

PageCast operates as a two-sided platform:

**For Readers (Audience):** Browse and purchase stories. Open in a secure browser reader. Read and listen with full audio immersion. Resume anytime.

**For Creators (Authors):** Write stories using a block-based editor. Assign AI or recorded voices to characters. Add music and sound effects. Publish and earn revenue.

### 0.5 Core Innovation (PBF Format Introduction)

PageCast invents a new proprietary file format:

> **PBF — PageCast Book File (`.pbf`)**

This is the equivalent of `.epub` for ebooks or `.mp4` for video — but purpose-built for cinematic audio-text storytelling. It is a secure, browser-native format combining structured HTML, synchronized audio, and encrypted production metadata.

### 0.6 Target Market & Use Cases

**Primary Markets:**

1. Children's storybooks 📚
2. Islamic stories and educational content 🕌
3. Language learning books 🌍
4. Romance and fiction indie authors ❤️
5. Comics and visual novels 🎭

Parents love audio-text synchronization. The first launch niche is children's Islamic stories — a perfect starting point with clear demand and underserved market.

## 3. PageCast Storytelling Format (PBF v1.0)

### 3.1 Overview

The **PageCast Book Format (PBF)** is PageCast's proprietary storytelling format — the platform's equivalent of EPUB for ebooks. It is a secure, browser-native, cinematic storytelling format combining structured HTML, synchronized audio, and encrypted production metadata.

The format is structured in two layers:

| Layer | Visible to Reader | Content |
|-------|------------------|---------|
| 🎭 Presentation Layer | Yes | HTML layout |
| 🎬 Production Layer | No | Audio, voices, sync, DRM |

Think of it like a movie: the viewer sees the film, but behind it is script, actors, and timeline.

**Design Goals — the format MUST:**
1. Work in any modern browser
2. Support cinematic audio storytelling
3. Keep content secure (commercial ready)
4. Allow reader personalization
5. Support both AI and human voices
6. Be easy for authors to produce
7. Be extensible for future animation and AI features

### 3.2 .pbf File Specification

**File extension (locked):** `.pbf` — PageCast Book File

Internally it is an **encrypted ZIP package**.

### 3.3 File Structure & Packaging System

```
book.pbf
│
├── manifest.json
├── story.html
├── production.json   (encrypted)
├── assets/
│   ├── audio/
│   ├── music/
│   ├── sfx/
│   ├── images/
│   └── cover.jpg
└── styles.css
```

### 3.4 manifest.json (Public Metadata)

Contains storefront and library information. This file is **NOT encrypted**.

```json
{
  "id": "book_001",
  "title": "The Whispering Forest",
  "author": "A. Rahman",
  "language": "en",
  "duration_minutes": 32,
  "chapters": 6,
  "has_music": true,
  "has_sfx": true,
  "version": "1.0"
}
```

### 3.5 production.json (Encrypted Timeline System)

Contains all hidden production data: audio file paths, voice IDs, sync timestamps, DRM watermark, and user token binding. This file is **encrypted with AES-256**.

**Audio timeline example:**

```json
{
  "timeline": [
    { "id": "n1", "audio": "audio/n1.mp3", "start": 0 },
    { "id": "d1", "audio": "audio/d1.mp3", "start": 4.2 },
    { "id": "pause_1", "duration": 2 },
    { "id": "sfx_1", "audio": "sfx/branch.mp3", "start": 6.5 }
  ]
}
```

HTML never contains timing. Separation keeps the presentation layer clean.

### 3.6 story.html (Presentation Layer)

This is what the reader sees. It uses custom semantic tags defined by the PBF specification.

**Allowed Tags:**

| Tag | Purpose |
|-----|---------|
| `<chapter>` | Chapter container |
| `<scene>` | Scene container |
| `<narration>` | Narrator text |
| `<dialogue>` | Character speech |
| `<thought>` | Internal thoughts |
| `<quote>` | Poem / Quran / letter |
| `<pause>` | Dramatic pause |
| `<sfx>` | Sound effect trigger |

### 3.7 Character Definition System

Defined inside `production.json`. Characters are required before dialogue can exist.

```json
{
  "characters": [
    {
      "id": "narrator",
      "role": "narrator",
      "defaultVolume": 1.0
    },
    {
      "id": "aisha",
      "role": "character",
      "displayName": "Aisha",
      "color": "#FF6B6B"
    }
  ]
}
```

Character objects enable:
- Automatic voice switching
- Per-character volume control by reader
- Future: character profile pages

### 3.8 Scene-Based Architecture Model

The book is **timeline-driven**, not page-driven.

**Hierarchy:**

```
Book
 └── Chapters
       └── Scenes
             └── Blocks (Narration / Dialogue / Thought / Quote)
```

### 3.9 Audio Layer System

Each scene supports multiple simultaneous audio layers:

| Layer | Loop | Optional |
|-------|------|---------|
| Narration | No | No |
| Character Voices | No | Yes |
| Background Music | Yes | Yes |
| Ambient Sound | Yes | Yes |
| Sound Effects | No | Yes |

All layers mix in real time in the browser. This makes the book feel alive.

### 3.10 Timing & Synchronization Model

All timing lives in `production.json`. IDs in `story.html` are **REQUIRED** for sync. The reader engine uses these IDs to know which block to highlight at any given moment during playback.

### 3.11 Block Types Specification

**Official Chapter Syntax:**

```html
<chapter id="ch1" title="A Strange Night">

  <scene id="scene1" ambience="forest-night.mp3" music="mystery.mp3">

    <narration id="n1">
      The night was silent. Too silent.
    </narration>

    <dialogue id="d1" character="aisha">
      Did you hear that?
    </dialogue>

    <pause duration="2s"></pause>

    <sfx file="branch_snap.mp3"></sfx>

    <dialogue id="d2" character="ali">
      We are not alone.
    </dialogue>

  </scene>
</chapter>
```

**Block Type Descriptions:**

- **Narration** — Normal paragraph storytelling
- **Dialogue** — Character speech, rendered as a speech bubble with character name and color
- **Thought** — Internal monologue, rendered in italics
- **Quote** — Letter, poem, Quran verse — centered with special font
- **Pause** — Dramatic silence, invisible to reader but timed in engine
- **SFX Trigger** — Fires a sound effect at a point in the timeline

### 3.12 Reader Modes (Reading / Audiobook / Cinematic)

Readers have three consumption modes:

**Mode 1 — Reading Mode**
Manual scrolling, optional audio. Like a normal ebook.

**Mode 2 — Audiobook Mode**
Auto-scroll ON, highlight ON, music OFF by default. Hands-free listening like a podcast.

**Mode 3 — Cinematic Mode**
Full screen, music and ambience ON, larger text, slower dramatic pacing. Maximum immersion.

Modes only change engine settings, not content.

## 5. Reader Application (Rendering Engine)

### 5.1 Reader App Overview

**Official Name:** PageCast Reader App — Book Rendering Engine Spec v1.0

The Reader App is the engine that turns an `.pbf` file into a cinematic reading experience in the browser. It is the player — like how Netflix has a video player or Kindle has an ebook reader.

**Goal:** Play PBF books smoothly, securely, and beautifully in any modern browser.

**Core Responsibilities:**
1. Open encrypted PBF books securely
2. Render HTML story beautifully
3. Synchronize text and audio perfectly
4. Provide reader personalization controls
5. Prevent content piracy
6. Work on mobile, tablet, and desktop

This is both a reader UI and a media engine.

### 5.2 Secure Book Loading Flow

Step-by-step process of opening a book:

1. User clicks book in library
2. Browser requests book session token
3. Backend returns encrypted `production.json` and signed audio URLs
4. Book decrypted in memory only
5. Reader starts playback

**Security rule:** No permanent files stored on device.

### 5.3 Rendering Engine Architecture

```
User Browser
   │
   ├── UI Layer (Reader Interface)
   ├── Rendering Engine (HTML Renderer)
   ├── Audio Engine (Web Audio API)
   ├── Timeline Sync Engine
   ├── DRM Security Layer
   └── Streaming & Decryption Layer
```

Five core engines work together to deliver the experience.

### 5.4 HTML Story Parsing System

The Book Loader Engine:
- Downloads `manifest.json`
- Loads `story.html` into a sandboxed iframe
- Fetches encrypted `production.json`
- Initializes character and timeline data

The story HTML runs inside a **sandboxed iframe** to prevent content copying.

**Tag rendering:**

| Tag | Rendered As |
|-----|-------------|
| `<narration>` | Paragraph block |
| `<dialogue>` | Speech bubble |
| `<thought>` | Italic block |
| `<quote>` | Centered block |
| `<pause>` | Invisible timing marker |

### 5.5 Audio Engine (Web Audio API Layering)

Uses the **Web Audio API** for real-time mixing of multiple audio layers simultaneously.

**Audio Channels:**

| Channel | Type |
|---------|------|
| Narrator | Voice track |
| Characters | Voice track |
| Music | Loop track |
| Ambience | Loop track |
| SFX | Triggered track |

All channels have independent volume sliders.

### 5.6 Timeline Synchronization Engine

This engine keeps text highlight synced with audio — the "magic" of PageCast.

**Workflow:**
1. Load timeline from `production.json`
2. Play narration audio
3. Highlight block by ID when audio starts
4. Auto-scroll to active block

The reader feels like the text is being read live.

### 5.7 Text Highlighting System

When audio plays:
- Current block **glows softly**
- Previous blocks **fade out**
- Page **auto-scrolls smoothly**

This gives a karaoke-style reading experience.

### 5.8 Auto-scroll & Immersive Mode

Auto-scroll is tied to audio timing. When active, the reader never needs to scroll manually. In Cinematic Mode, full-screen and dramatic pacing create maximum immersion.

### 5.9 Reader Modes Implementation

**📖 Reading Mode**
- Audio optional
- Manual scroll
- Highlight disabled by default

**🎧 Audiobook Mode**
- Auto-scroll ON
- Highlight ON
- Music OFF by default

**🎬 Cinematic Mode**
- Fullscreen
- Music + ambience ON
- Larger text
- Slower dramatic pacing

Modes only change engine settings, not content.

### 5.10 User Control Panel System

**Floating bottom panel:**

*Audio Controls:*
- Master volume
- Narrator volume
- Character volume
- Music volume
- SFX volume
- Playback speed

*Reading Controls:*
- Font size
- Theme (Light / Dark / Sepia)
- Dyslexia font toggle
- Toggle auto-scroll

All preferences saved per user.

### 5.11 Resume & Bookmark System

Reader App saves:
- Last block read
- Playback timestamp
- Reader preferences

When user reopens book: **Resume instantly.**

### 5.12 Streaming Optimization System

Audio is streamed per scene — not fully downloaded upfront.

**Why:** Books may contain 100+ audio files.

**Engine strategy:**
- Load current scene audio
- Preload next scene audio in background

This ensures:
- Fast loading
- Low bandwidth usage
- Smooth uninterrupted playback

### 5.13 Accessibility Features

Reader can enable:
- Audio-only mode
- Closed captions
- High contrast theme
- Dyslexia font
- Adjustable playback speed

Accessibility is part of the core engine, not optional.

### 5.14 DRM & Security Layer (Soft DRM MVP + Future Hard DRM)

**Anti-piracy features:**
- Sandboxed HTML iframe
- Disable right-click and text selection
- Obfuscated DOM structure
- Streaming signed URLs
- Audio served as chunks (not full downloadable files)
- Per-user watermark injection

**Goal:** Prevent casual piracy while maintaining smooth reader experience.

**Performance Targets:**
- Open book in < 2 seconds
- Start audio in < 500ms
- Scroll at 60fps
- Work on mid-range phones

### 6.9 DRM & Streaming Service

When a reader opens a book, the backend generates:
- A session token
- A decryption key
- Signed audio URLs (expiring)

Audio is streamed in chunks — preventing downloading of full files. This connects to the Reader Engine's DRM layer.

### 6.11 Revenue Distribution Model

### 11.2 Cost Avoidance Strategy

**Monolith over microservices:** One NestJS monolith instead of separate services eliminates infrastructure cost entirely at launch.

**Shared frontend app:** Reader App and Author Studio deployed in the same Next.js project on Vercel.

**Stripe is free upfront:** No monthly fee — only takes a percentage per successful transaction, meaning zero cost before first sale.

### 11.3 Soft DRM Strategy (MVP Version)

True DRM (encryption streaming + device locking) costs money. For MVP:

**We KEEP:**
- Signed URLs (no public file links)
- Watermarking text per user

**We DELAY:**
- Advanced encryption streaming
- Offline mode
- Hardcore anti-piracy systems

This is acceptable for MVP. The goal is to prove market demand, not build fortress-level security.

### 11.4 BYO AI Voice Strategy (No Cost TTS)

AI voice is the only significant variable cost risk. MVP solution:

**Authors can connect their own TTS API key:**
- OpenAI TTS
- ElevenLabs
- Google TTS

The platform does not pay for generation during MVP. Authors bring their own key. Smart and safe.

### 13.2 File Format Reference (.pbf Full Spec Summary)

The `.pbf` file is an **encrypted ZIP archive** containing:

| File | Purpose | Encrypted |
|------|---------|-----------|
| `manifest.json` | Public book metadata | No |
| `story.html` | Presentation layer (story content) | No (sandboxed) |
| `production.json` | Timeline, voices, DRM data | Yes (AES-256) |
| `assets/audio/` | Narration and dialogue audio files | Private |
| `assets/music/` | Background music tracks | Private |
| `assets/sfx/` | Sound effects | Private |
| `assets/images/` | Illustrations | Private |
| `assets/cover.jpg` | Cover image | No |
| `styles.css` | Typography and visual styling | No |

### 13.3 System Diagrams (Architecture Maps)

**Book Loading Flow:**
```
User clicks book
    → Request session token from backend
    → Backend returns decryption key + signed audio URLs
    → Book decrypted in memory
    → story.html loaded in sandboxed iframe
    → production.json parsed for timeline
    → Audio engine initialized
    → Playback begins
```

**Author Publish Flow:**
```
Author clicks Publish
    → Studio generates story.html
    → Studio generates production.json
    → Assets packaged
    → Production layer encrypted
    → .pbf file created
    → Uploaded to S3/R2
    → Book listed in store
```

**Reader Purchase Flow:**
```
Reader clicks Buy
    → Stripe checkout session created
    → Reader completes payment
    → Stripe webhook fires
    → Purchase record created in database
    → Book added to reader's library
    → Reader opens book
```
